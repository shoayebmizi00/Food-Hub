# Render Environment Variables

Use these for the Dockerized backend service on Render. Do not add backend secrets to Vercel.

| Variable | Required | Description |
| --- | --- | --- |
| NODE_ENV | Yes | Set to `production` on Render. |
| PORT | No | Render injects this automatically; `10000` is a safe Docker default. |
| DATABASE_URL | Yes | Supabase PostgreSQL connection string used by Prisma. |
| JWT_SECRET | Yes | Long random secret for signing access tokens. Can be generated by Render. |
| JWT_EXPIRES_IN | Yes | Access token lifetime, for example `7d`. |
| REFRESH_TOKEN_SECRET | No | Future refresh-token signing secret. Generate before enabling refresh tokens. |
| REFRESH_TOKEN_EXPIRES_IN | No | Future refresh-token lifetime, for example `30d`. |
| FRONTEND_URL | Yes | Primary Vercel frontend origin, for example `https://your-app.vercel.app`. |
| ALLOWED_ORIGINS | Yes | Comma-separated CORS origins, usually Vercel plus local dev if needed. |
| COOKIE_SECRET | No | Future signed-cookie secret. Generate before enabling signed cookies. |
| DEV_OTP | No | Development-only OTP fallback. Leave blank or unset in production email flows. |
| LOG_LEVEL | No | Logging level, for example `info`. |
| SEED_USER_PASSWORD | Required for `prisma db seed` | Password for demo seeded users. Do not use a shared public value. |
| SUPER_ADMIN_EMAIL | Required for production seed | Email for the seeded super admin account. |
| SUPER_ADMIN_PASSWORD | Required for production seed | Password for the seeded super admin account. |
| SUPER_ADMIN_NAME | Required for production seed | Display name for the seeded super admin account. |
| SMTP_HOST | No | SMTP host for future email delivery. |
| SMTP_PORT | No | SMTP port, commonly `587`. |
| SMTP_USER | No | SMTP username. |
| SMTP_PASSWORD | No | SMTP password. |
| SMTP_FROM | No | Sender email address. |
| CLOUDINARY_CLOUD_NAME | No | Cloudinary cloud name for future uploads. |
| CLOUDINARY_API_KEY | No | Cloudinary API key. |
| CLOUDINARY_API_SECRET | No | Cloudinary API secret. |
| STRIPE_SECRET_KEY | No | Stripe secret key for future Stripe payments. |
| STRIPE_WEBHOOK_SECRET | No | Stripe webhook signing secret. |
| SSLCOMMERZ_STORE_ID | No | SSLCommerz store ID for future payments. |
| SSLCOMMERZ_STORE_PASSWORD | No | SSLCommerz store password. |
| SSLCOMMERZ_IS_LIVE | No | `true` for live SSLCommerz, `false` for sandbox. |
| BKASH_USERNAME | No | bKash API username. |
| BKASH_PASSWORD | No | bKash API password. |
| BKASH_APP_KEY | No | bKash app key. |
| BKASH_APP_SECRET | No | bKash app secret. |
| NAGAD_API_KEY | No | Nagad API key. |
| NAGAD_API_SECRET | No | Nagad API secret. |
| NAGAD_MERCHANT_ID | No | Nagad merchant ID for merchant API integrations. |
| NAGAD_MERCHANT_PRIVATE_KEY | No | Nagad merchant private key for merchant API integrations. |

## Vercel Frontend Variables

| Variable | Required | Description |
| --- | --- | --- |
| VITE_API_URL | Yes | Render backend API base URL: `https://food-hub-xg61.onrender.com/api`. |

This is a Vite app, so only variables prefixed with `VITE_` are exposed to frontend code.
